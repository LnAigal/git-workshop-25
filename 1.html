1) Git

About Git
Git is a distributed version control system that tracks changes in your files and lets you move between versions safely.

Setting up Git

git --version
git config --global user.name "Your Name"
git config --global user.email "you@example.com"

2) Version Control

You can go back to previous project states and recover earlier changes.

Each commit is a checkpoint you can return to later.

3) Git Commands

init Initialize repository 
git init
status See changes 
git status
add Stage files 
git add .
commit Commit changes 
git commit -m "Your message here"
log See commits 
git log --oneline
ignore Ignore files 
.gitignore

5) Git Branches

git branch         # list branches
git branch -M b1    # create new branch
git switch -c b2    # create & switch to branch
git checkout        # switch branch

6) Hashes & Heads

Heads allow you to go back to previous commits easily using commit hashes.

7) Stash

git stash         # save staged changes to stack
git stash list    # view stashed changes
git stash apply   # apply latest stash

Stash acts like a stack: you can save multiple changes and reapply them later.
